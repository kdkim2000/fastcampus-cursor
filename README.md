# 개발자에게 배우는 Cursor AI 고급 노하우  
### — Agent 중심 AI Coding 실무 교재 —

이 저장소는 **Fastcampus 강의 「개발자에게 배우는 Cursor AI 고급 노하우」** 의  
공식 교재를 Markdown 형태로 정리한 자료입니다.

단순히 AI 도구 사용법을 나열하는 것이 아니라,  
**AI를 ‘도구’가 아닌 ‘동료이자 Agent’로 활용하는 실무 관점** 에서  
개발자의 사고방식, Workflow, 조직 적용까지 단계적으로 다룹니다.

---

## 📌 이 교재의 목표

- “AI를 써본 개발자”가 아니라 **“AI와 함께 일하는 개발자”** 가 된다
- Cursor를 중심으로 **Agent 기반 개발 Workflow** 를 설계할 수 있다
- 개인 생산성을 넘어 **조직 단위 AI 활용 구조** 를 이해한다
- AI 시대에 개발자의 역할이 어떻게 진화하는지 명확히 인식한다

## 🧭 Part별 핵심 요약 & 체크리스트

### ✅ Part1. AI 생태계 톺아보기

**핵심 요약**
- AI는 더 이상 단순한 자동완성 도구가 아니다
- Chat 기반 활용에서 Agent 기반 협업으로 패러다임이 이동했다
- Vibe Coding이 실무에서 실패하는 구조적 이유를 이해한다

**체크리스트**
- [ ] Chat과 Agent의 차이를 설명할 수 있는가?
- [ ] Context가 왜 중요한지 이해했는가?
- [ ] Code Agent가 왜 등장했는지 설명할 수 있는가?

---

### ✅ Part2. 실무에 AI Coding 적용하기

**핵심 요약**
- AI Coding의 가장 큰 리스크는 ‘보안’이다
- Cursor는 단순 IDE가 아니라 **Agent 실행 환경** 이다
- Rules와 MCP는 Cursor 활용의 핵심 축이다

**체크리스트**
- [ ] AI 활용 시 보안 리스크를 설명할 수 있는가?
- [ ] Cursor의 Rules와 MCP 개념을 이해했는가?
- [ ] 요구사항 기반으로 Cursor 환경을 구성할 수 있는가?

---

### ✅ Part3. 업무에 꼭 필요한 AI 고급 활용

**핵심 요약**
- AI는 개인 생산성 도구가 아니라 **업무 위임 대상** 이다
- Workflow를 분해하지 않으면 AI는 일을 못 한다
- 조직에서 AI를 쓰려면 표준화가 필수다
- Cursor가 불가능한 환경에서도 대안은 존재한다

**체크리스트**
- [ ] 나의 업무 Workflow를 설명할 수 있는가?
- [ ] Rules와 MCP로 Workflow를 구현할 수 있는가?
- [ ] 조직 관점에서 AI 활용의 위험 요소를 알고 있는가?
- [ ] Cursor 대체 오픈소스 도구의 장단점을 이해했는가?

---

### ✅ Part4. AI시대 개발자의 역량

**핵심 요약**
- AI 시대 개발자의 역할은 ‘코더’에서 ‘오케스트레이터’로 이동한다
- AI를 활용해 지식 습득과 문서 생산 속도를 극적으로 높일 수 있다
- 핵심 역량은 코드량이 아니라 **설계·판단·위임 능력** 이다

**체크리스트**
- [ ] AI Orchestrator 개념을 이해했는가?
- [ ] AI를 활용해 논문·문서를 다룰 수 있는가?
- [ ] AI 시대에 나의 성장 방향을 설명할 수 있는가?

## ✨ 에필로그 — AI 시대, 개발자는 사라지지 않는다

AI는 개발자를 대체하지 않는다.  
하지만 **AI와 함께 일할 줄 모르는 개발자** 는 빠르게 도태된다.

이 교재의 목적은  
“AI를 잘 쓰는 방법”을 알려주는 것이 아니라,

> **AI와 함께 일하는 개발자로 진화하는 방법** 을  
> 구조적으로 이해시키는 것이다.

이 저장소의 내용이  
여러분의 개발 인생에서 **하나의 전환점** 이 되기를 바랍니다.
