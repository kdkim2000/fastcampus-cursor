# Chapter 5. 개발에 활용 가능한 AI 도구

이 장에서는 실제 개발 현장에서
**어떤 AI 도구를 어떤 목적에 사용해야 하는지**를 정리한다.

중요한 점은:
- “어떤 도구가 더 좋다”가 아니라
- **“어떤 상황에서 어떤 도구가 적합한가”** 이다.

본 강의에서 다루는 주요 도구는 다음 네 가지이다.

- Claude
- Cursor
- V0
- CodeRabbit

이들은 서로 경쟁 관계가 아니라,
**서로 다른 역할을 맡는 도구들**이다.

---

## 5.1 Claude

### 5.1.1 Claude란 무엇인가

**Claude**는 Anthropic에서 개발한 대형 언어 모델(LLM) 기반 AI이다.

개발 관점에서 Claude의 가장 큰 특징은:
- 긴 컨텍스트 처리 능력
- 자연스러운 언어 이해
- 복잡한 요구사항 정리 능력

Claude는 IDE보다는
**사고·설계·정리 영역**에 강하다.

---

### 5.1.2 Claude의 강점

Claude가 특히 강한 영역은 다음과 같다.

- 요구사항 분석
- 설계 문서 작성
- 코드 리뷰 코멘트 생성
- 긴 문맥의 대화 유지

즉, Claude는
**“생각을 함께 정리해 주는 파트너”** 에 가깝다.

---

### 5.1.3 Claude를 언제 쓰는가

다음과 같은 상황에서 Claude가 적합하다.

- 기능 구현 전에 구조를 잡고 싶을 때
- 복잡한 비즈니스 로직을 설명해야 할 때
- 코드 리뷰 관점에서 의견이 필요할 때
- 문서화 작업이 필요할 때

👉 Claude는 **사전 사고 정리**에 최적화된 도구이다.

---

## 5.2 Cursor

### 5.2.1 Cursor란 무엇인가

**Cursor**는 AI 기능이 깊게 통합된 IDE이다.

기존 IDE에 AI를 “붙인” 것이 아니라,
- Agent 개념
- Context 관리
- 프로젝트 단위 작업
을 전제로 설계되었다.

---

### 5.2.2 Cursor의 강점

Cursor의 핵심 강점은 다음과 같다.

- 코드베이스 전체를 맥락으로 활용
- 파일 간 이동과 수정에 능함
- Agent 기반 작업 수행
- Rules를 통한 행동 통제

즉, Cursor는
**“실제로 코드를 바꾸는 도구”** 이다.

---

### 5.2.3 Cursor를 언제 쓰는가

Cursor는 다음 상황에서 가장 강력하다.

- 실제 코드 작성 및 수정
- 리팩토링 작업
- 여러 파일에 걸친 기능 구현
- Agent 기반 작업 위임

👉 Claude가 설계자라면,
👉 Cursor는 **실행자 IDE**이다.

---

## 5.3 V0

### 5.3.1 V0란 무엇인가

**V0**는 UI 생성에 특화된 AI 도구이다.

- 주로 React 기반 UI 생성
- 디자인 → 코드 변환
- 빠른 시각적 결과 제공

---

### 5.3.2 V0의 강점

V0의 가장 큰 강점은 속도이다.

- 화면 단위 UI 즉시 생성
- 컴포넌트 구조 자동 설계
- 반복적인 UI 작업 최소화

👉 프론트엔드 초기 개발에 매우 효과적이다.

---

### 5.3.3 V0를 언제 쓰는가

V0는 다음과 같은 경우에 적합하다.

- 초기 UI 프로토타이핑
- 관리자 페이지 구성
- 디자인 시안 빠른 구현

단, 생성된 코드는
반드시 프로젝트 구조에 맞게 조정해야 한다.

---

## 5.4 CodeRabbit

### 5.4.1 CodeRabbit이란 무엇인가

**CodeRabbit**은
PR(Pull Request) 기반 AI 코드 리뷰 도구이다.

- GitHub 연동
- 변경 사항 분석
- 리뷰 코멘트 자동 생성

---

### 5.4.2 CodeRabbit의 강점

CodeRabbit은 다음에 강하다.

- 일관된 리뷰 기준 유지
- 반복 리뷰 자동화
- 휴먼 리뷰어의 부담 감소

특히 팀 단위 개발에서
큰 효과를 발휘한다.

---

### 5.4.3 CodeRabbit을 언제 쓰는가

- PR 리뷰 속도를 높이고 싶을 때
- 리뷰 기준을 표준화하고 싶을 때
- 코드 품질을 일정 수준 이상 유지하고 싶을 때

👉 CodeRabbit은
**“팀 생산성을 높이는 AI”** 이다.

---

## 5.5 도구 간 역할 정리

각 도구의 역할을 한눈에 정리하면 다음과 같다.

| 도구 | 주요 역할 |
|----|---------|
| Claude | 사고 정리, 설계, 문서 |
| Cursor | 코드 실행, 수정, Agent 작업 |
| V0 | UI 생성 |
| CodeRabbit | 코드 리뷰 |

이 장의 핵심은:
> **모든 것을 하나의 도구로 해결하려 하지 말라**는 것이다.

---

## 5.6 Chapter 5 정리

- AI 도구는 역할에 따라 선택해야 한다
- Claude는 사고 파트너이다
- Cursor는 Agent 중심 IDE이다
- V0는 UI 생산성을 높인다
- CodeRabbit은 팀 리뷰를 자동화한다
- 도구 조합이 곧 생산성 전략이다
