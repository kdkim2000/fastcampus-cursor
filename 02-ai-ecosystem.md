# Chapter 2. AI 생태계 톺아보기

본 장에서는 현재 우리가 사용하는 AI 코딩 도구들이
어떤 흐름과 문제의식 속에서 발전해 왔는지를 살펴본다.

AI는 어느 날 갑자기 등장한 “마법의 도구”가 아니라,
개발 생산성을 높이기 위한 도구들이
**점진적으로 진화한 결과물**이다.

이 흐름을 이해해야만,
왜 지금 Agent와 Cursor가 주목받는지 명확히 이해할 수 있다.

---

## 2.1 AI 개발 도구의 진화

AI 코딩 도구의 진화는 다음과 같은 단계로 정리할 수 있다.

> **Lint → Static Analysis → Copilot → Agent**

이 흐름은 단순히 기술의 발전이 아니라,  
**“개발자가 무엇을 AI에게 맡기기 시작했는가”**의 역사이기도 하다.

---

### 2.1.1 Lint: 규칙 기반 자동화의 시작

가장 초기의 개발 자동화 도구는 **Lint**였다.

- 문법 오류 탐지
- 코드 스타일 검사
- 정해진 규칙(rule)에 따른 경고 출력

Lint의 특징은 다음과 같다.

- 사람이 미리 정의한 규칙만 검사 가능
- 코드의 “의미”는 이해하지 못함
- 빠르고 안정적이지만 유연성은 낮음

👉 이 단계에서 자동화의 역할은  
**“개발자가 실수하지 않도록 감시하는 도구”**에 가까웠다.

---

### 2.1.2 Static Analysis: 의미를 이해하려는 시도

다음 단계는 **정적 분석(Static Analysis)** 이다.

- 코드 실행 없이 흐름 분석
- 타입 추론
- 잠재적 버그 탐지

정적 분석 도구는 Lint보다 한 단계 발전하여:
- 코드 구조
- 데이터 흐름
- 의존성 관계
를 부분적으로 이해하기 시작했다.

하지만 여전히 한계가 있었다.

- 규칙 기반에서 크게 벗어나지 못함
- 도메인 맥락은 이해 불가
- 코드 “생성”은 불가능

👉 이 단계에서도 AI는 **분석자(Reviewer)** 역할에 머물렀다.

---

### 2.1.3 Copilot: 코드 생성을 시작하다

Copilot의 등장은 큰 전환점이었다.

- 자연어 기반 코드 생성
- 함수 단위 보완
- 개발자 타이핑을 대체

Copilot은 다음과 같은 변화를 가져왔다.

- AI가 **코드를 ‘읽는 것’에서 ‘쓰는 것’으로 이동**
- 개발자는 작성자에서 편집자 역할로 변화
- 생산성의 체감적 향상

그러나 Copilot에도 명확한 한계가 있었다.

- 파일 단위, 함수 단위에 머무름
- 프로젝트 전체 맥락 이해 부족
- 여러 작업을 스스로 계획하지 못함

👉 Copilot은 훌륭한 **보조 도구(Assistant)** 였지만,
아직 “일을 맡길 수 있는 존재”는 아니었다.

---

### 2.1.4 Agent: 일을 맡길 수 있는 존재

Agent는 Copilot과 근본적으로 다르다.

- 작업을 **계획(Plan)** 하고
- 여러 파일을 **탐색**하며
- 반복적으로 **실행–검증–수정**을 수행한다

즉, Agent는:
- 단순 코드 생성기가 아니라
- **업무 단위(Task)를 수행하는 주체**이다.

이 단계에서 AI의 역할은:
- 보조(Assistant) → 실행자(Agent) 로 전환된다.

👉 이 변화가 바로 **현재 AI 개발 도구 생태계의 핵심**이다.

---

## 2.2 Agent 시대의 도래

Agent가 주목받는 이유는 단순히 기술이 좋아졌기 때문이 아니다.
**산업 구조 자체가 Agent를 요구하고 있기 때문**이다.

---

### 2.2.1 Sam Altman의 발언

OpenAI의 CEO Sam Altman은 여러 인터뷰와 발표에서
다음과 같은 메시지를 반복해서 전달했다.

- “The future is agents”
- “AI will do real work”
- “AI will be a form of labor”

이 발언의 핵심은 단순하다.

> AI는 더 이상 도구(tool)가 아니라  
> **노동력(labor)** 이 된다는 것이다.

---

### 2.2.2 AI 노동력 편입의 의미

AI가 노동력으로 편입된다는 것은 다음을 의미한다.

- AI에게 “작업 단위”를 맡긴다
- 결과물을 검토하고 책임은 사람이 진다
- 반복 작업, 탐색 작업, 보조 판단을 AI가 수행한다

개발자의 역할은 자연스럽게 변화한다.

| 기존 개발자 | Agent 시대 개발자 |
|------------|------------------|
| 직접 구현 | 방향 제시 |
| 모든 코드 작성 | 코드 리뷰 |
| 세부 로직 고민 | 구조와 의사결정 |

👉 개발자는 점점 **실행자**에서  
**설계자·의사결정자**로 이동하게 된다.

---

## 2.3 AI 생태계 관점에서 Cursor의 위치

이러한 흐름 속에서 Cursor는 다음 위치에 있다.

- IDE 기반
- Agent 중심
- Context를 시각적으로 관리
- 개발자 워크플로우에 자연스럽게 통합

즉, Cursor는:
- Copilot 이후의 도구이며
- Agent 시대를 전제로 설계된 IDE이다.

이후 챕터에서는:
- 왜 Cursor가 Agent에 적합한지
- Context를 어떻게 설계해야 하는지
- Rules와 MCP는 왜 필요한지
를 단계적으로 다룬다.

---

## 2.4 Chapter 2 정리

- AI 개발 도구는 **점진적으로 진화**해 왔다
- Lint → Static Analysis → Copilot → Agent 흐름을 이해해야 한다
- Agent는 “코드를 쓰는 AI”가 아니라 “일을 하는 AI”이다
- AI는 이제 **노동력의 일부**로 편입되고 있다
- Cursor는 이 Agent 시대를 전제로 만들어진 도구이다
