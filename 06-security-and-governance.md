# Chapter 6. AI 활용 보안과 거버넌스

AI 도구는 생산성을 극적으로 높여 주지만,
동시에 **조직 차원의 보안 리스크** 를 동반한다.

특히 개발 영역에서의 AI 활용은:
- 소스 코드
- 인프라 설정
- 비즈니스 로직
과 같은 **민감 정보** 와 직접적으로 연결된다.

이 장에서는 AI를 **안전하게, 지속 가능하게** 활용하기 위한
보안과 거버넌스 관점을 다룬다.

---

## 6.1 Outbound / Inbound Risk

AI 활용 리스크는 크게 두 가지로 나눌 수 있다.

- Outbound Risk
- Inbound Risk

이 두 가지를 구분하지 못하면
보안 정책을 설계할 수 없다.

---

### 6.1.1 Outbound Risk (정보 유출 리스크)

**Outbound Risk** 란,
내부 정보가 AI를 통해 외부로 유출되는 위험을 의미한다.

대표적인 예시는 다음과 같다.

- 소스 코드 전체를 AI에게 그대로 전달
- 인증 정보, API Key 포함된 코드 입력
- 고객 데이터, 로그, 설정 파일 공유

AI는 의도가 없지만,
**입력된 정보는 외부 시스템으로 전송** 된다.

👉 즉, “AI에게 물어봤다”는 행위 자체가
정보 반출이 될 수 있다.

---

### 6.1.2 Inbound Risk (위험 코드 유입)

**Inbound Risk** 는 그 반대 방향의 위험이다.

- AI가 생성한 코드에 취약점 포함
- 라이선스 문제가 있는 코드 생성
- 보안 규칙을 위반한 구현

특히 다음 상황에서 위험이 커진다.

- 검증 없이 코드 복사·붙여넣기
- “AI가 만들었으니 맞겠지”라는 착각
- 리뷰 프로세스 생략

👉 AI 코드는 **검증되지 않은 외부 코드** 와 같다.

---

## 6.2 최소 권한 전략

보안을 강화하기 위해 가장 중요한 원칙은
**최소 권한(Least Privilege)** 이다.

AI에게도 이 원칙은 그대로 적용된다.

---

### 6.2.1 최소 권한 전략이란

최소 권한 전략이란:
- 꼭 필요한 정보만 제공하고
- 꼭 필요한 작업만 허용하는 것이다.

AI에게 모든 것을 보여줄 이유는 없다.

---

### 6.2.2 AI에 적용하는 최소 권한

실무에서는 다음과 같은 방식으로 적용한다.

- 작업 대상 파일만 Context로 제공
- 민감한 설정 파일 제외
- 테스트 코드부터 위임
- 핵심 로직은 직접 구현 또는 리뷰 강화

👉 AI는 “전지전능한 존재”가 아니다.
권한을 제한할수록 결과는 안정적이다.

---

### 6.2.3 권한 분리와 역할 분담

AI를 하나의 계정처럼 취급하면 안 된다.

- 설계용 AI
- 구현용 AI
- 리뷰용 AI

역할을 분리하면:
- 리스크가 분산되고
- 사고 발생 시 영향 범위가 줄어든다.

---

## 6.3 Cursor & Claude 보안 정책

도구 자체가 제공하는 보안 정책을 이해하는 것도 매우 중요하다.

---

### 6.3.1 Cursor 보안 정책

Cursor는 IDE 레벨에서 다음과 같은 보안 기능을 제공한다.

- **Privacy Mode**
  - 입력된 코드가 학습 데이터로 사용되지 않도록 설정 가능
- **Permission Control**
  - 파일 접근 및 수정 권한 제어
- **Local Context 중심 설계**
  - 불필요한 데이터 전송 최소화

Cursor를 사용할 때는 반드시:
- Privacy Mode 설정 여부를 확인하고
- 프로젝트 단위 Rules를 정의해야 한다.

---

### 6.3.2 Claude 보안 정책

Claude는 다음과 같은 보안 원칙을 따른다.

- 기본적으로 사용자 데이터 학습 제한
- 엔터프라이즈 환경에서 데이터 격리 제공
- 장기 컨텍스트 사용 시 명시적 입력 필요

하지만:
- 사용자가 입력한 정보는 여전히 민감할 수 있다.

👉 Claude는 안전한 도구이지만,
**사용자의 판단이 보안을 완성** 한다.

---

## 6.4 조직 관점의 거버넌스

개인 사용을 넘어,
조직 차원의 AI 활용에는 거버넌스가 필요하다.

- AI 사용 가이드라인 정의
- 허용 도구 목록 관리
- 코드 리뷰 필수화
- 로그 및 감사 체계 구축

거버넌스 없는 AI 활용은
“편리하지만 위험한 도구”가 된다.

---

## 6.5 Chapter 6 정리

- AI 활용에는 Outbound / Inbound Risk가 존재한다
- AI는 항상 최소 권한 원칙으로 다뤄야 한다
- AI 생성 코드는 반드시 검증해야 한다
- Cursor와 Claude의 보안 정책을 이해해야 한다
- 조직 차원의 거버넌스가 필수이다
